# 프론트엔드 UI 구현 TODO

## 1. 생성형 채팅 UI
- 자유로운 대화형 인터페이스
- 사용자가 자연어로 질문하면 AI가 응답
- 예: "삼겹살로 뭐 해먹지?", "간단한 저녁 추천해줘"
- 대화 히스토리 표시
- 스트리밍 응답 지원 (타이핑 효과)

### 주요 기능
- 채팅 입력창
- 메시지 버블 (사용자/AI)
- 추천 결과에서 레시피 보기, 장바구니 담기 액션

---

## 2. 일반검색 UI

### 2-1. 요리추천
- 카테고리 선택 (한식, 중식, 일식, 양식)
- 조건 입력 (예: "간단한 저녁", "매운 요리")
- 선택/입력 기반으로 요리 목록 추천

### 2-2. 레시피검색
- 요리명 직접 검색 (예: "김치찌개")
- 검색 결과: 조리시간, 난이도, 필요한 재료
- 레시피 상세 보기

### 공통 기능
- 검색 결과 리스트
- 레시피 보기 버튼
- 장바구니 담기 버튼

---

## 3. 스마트추천 UI

### 3-1. 할인상품으로 뭐 해먹지?
- 오늘의 할인상품 목록 표시 (마트 API 연동)
- 관심 상품 선택 (체크박스)
- 직접 입력으로 추가 가능
- 선택한 할인상품 기반 레시피 추천
- 할인 가격 정보 함께 표시

### 3-2. 냉장고 털기
- 냉장고에 있는 재료 선택/입력
- 선택된 재료 실시간 표시
- 보유 재료 기반 레시피 추천
- 부족한 재료 중 할인 중인 것 안내

### 공통 기능
- 재료/상품 선택 UI (체크박스 + 직접입력)
- AI 추천 결과
- 할인 정보 연계

## 파일 구조 (예정)
```
app/front/static/
├── index.html          # 메인 진입점
├── css/
│   └── style.css       # 공통 스타일
├── js/
│   ├── app.js          # 공통 로직
│   ├── chat.js         # 생성형 채팅
│   ├── search.js       # 일반검색
│   └── smart.js        # 스마트추천
└── pages/
    ├── chat.html       # 생성형 채팅 페이지
    ├── recommend.html  # 요리추천 페이지
    ├── recipe.html     # 레시피검색 페이지
    ├── discount.html   # 할인상품 추천 페이지
    └── fridge.html     # 냉장고 털기 페이지
```
